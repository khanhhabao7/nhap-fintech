📖 Fun(d) Raising - Game Guide

1. Overview

Fun(d) Raising is a multiplayer startup simulation game where 2–10 players compete to build the most attractive project. Each player creates a startup, chooses a custom deck of cards, and navigates random events. Investors (our system) decide where to put their money. The host controls the game flow. After several rounds (phases), a final score determines the winner.

2. Setup

2.1 Host creates a room

The host enters the number of players (2–10) and clicks Create Room. The system generates unique join links – one per player. The host sends each link to the corresponding player.

2.2 Players enter project details

Each player opens their link and fills in the required information about their startup (price, costs, production forecast, capital structure, etc.). All fields have reasonable limits to keep the game balanced.

2.3 Players build their deck

After submitting the project, each player selects:

  • 19 Active Cards (mandatory) – three types:
    • Aggressive: boosts Hype, often lowers Transparency.
    • Defensive: increases Transparency, reduces legal risk.
    • Capital: affects funding, trust, or financial structure.
  • Up to 3 Reaction Cards: one‑time cards that trigger automatically when a specific condition is met.

Note: Cards are selected by clicking on them (no "highlight" step). The host decides when the deck selection phase ends and the game starts.

3. Game Flow – Phases

The number of phases depends on the chosen scale:

  • Small → 5 phases
  • Medium → 7 phases
  • Large → 9 phases

The host controls the progression: after each phase, the host decides when to move to the next one.

During a phase:

a. Random event (Scenario)
Each project receives a random event, including market, internal, external, or regulatory events.

b. Player actions
Each player sees 5 Active cards drawn randomly from their deck. They may play Active cards (costs energy – 3 energy per phase) by clicking on the card and then clicking the Play Card button. They may Mulligan once per phase (discard hand, draw 3 new cards) – costs 1 energy, lowers Transparency by 2 and trust of all bots by 1. Reaction cards that are triggered will appear as clickable buttons; they can be used at any time during the phase (each only once). The host decides when the phase ends and proceeds to the next one.

c. Host runs the phase
When the host clicks Run Phase, the system:

  • Applies all played cards and event effects.
  • Updates project stats (funding progress, runway, etc.).
  • Bots (200 bots of 4 types: FOMO, Value Hunter, Whale, Random) evaluate all active projects.
  • Bots may withdraw money from less attractive projects.
  • Bots may invest new money into attractive projects (softmax distribution).
  • Bots have short‑term memory decay, they remember past attractiveness, so a project that performed poorly before may struggle to regain trust.
  • The phase ends. If a project runs out of cash, it goes bankrupt and is eliminated.

4. End of Game & Winner

The game ends when all projects have finished their maximum number of phases (or are bankrupt). The host sees a final ranking based on Final Score (0–100). The score takes into account funding progress, speed, expected ROI, transparency, and a penalty for unrealistic valuation. The project with the highest score wins.

5. Quick Tips

  • Balance Aggressive (Hype) and Defensive (Transparency) cards – too much hype hurts credibility.
  • Save energy for impactful cards; don't waste it on weak effects.
  • Choose Reaction cards that fit your strategy – they can save you in critical moments (e.g., bot withdrawals, bad events).
  • Value Hunter bots focus on intrinsic value (margin, burn rate, growth, revenue scale, capital efficiency, leverage) – build a solid financial foundation to attract them.
